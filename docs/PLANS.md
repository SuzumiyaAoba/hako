# 今後の計画

目的: 最低限のアプリ機能（ノート閲覧・リンク/バックリンク・検索・グラフ）を揃えつつ、
API/フロントをレビューしやすい粒度で段階的に実装する。

## タスク一覧（PR想定）

1. OpenAPI ドキュメントの基本整備
   - `/openapi.json` の出力を確認できる最小構成を確定
   - タイトル/バージョン/説明などのメタ情報を追加

2. ルーティングのバリデーション整理
   - `@hono/standard-validator` に統一
   - 既存エンドポイントの入力スキーマを追加

3. レスポンススキーマの拡充
   - `notes`/`notes/:id` のレスポンス定義を厳密化
   - エラー応答（400/404）の共通スキーマ化

4. ルート定義の整理
   - OpenAPI 用の `describeRoute` を一箇所に集約
   - ルートの責務分割（例: `routes/notes.ts`）

5. フロントエンドの基盤整備
   - Next.js ルーティング雛形（一覧/詳細）を作成
   - API クライアント層を追加（`/notes`, `/notes/:id`）

6. Markdown 表示の最小実装
   - `unified` パイプライン（remark/rehype）を導入
   - Markdown を HTML として表示

7. Wiki link の最低限対応
   - `[[Title]]` をリンクとして表示（解決失敗はスタイル分け）
   - ノート詳細にバックリンク一覧を表示

8. 検索の最低限対応
   - タイトル検索（まずは前方一致/部分一致）
   - 結果からノート詳細へ遷移

9. グラフ表示の初期版
   - D3 の簡易グラフを表示
   - クリックでノート詳細へ遷移

10. テスト整備

- バリデーション成功/失敗のケース追加
- OpenAPI 生成のスナップショット（または形状検証）追加
- フロントの主要ページのスモークテスト

11. ドキュメント更新

- `docs/` に API 利用例を追記
- 開発手順（バリデーション/生成物の確認方法）を追記
- フロントの画面構成と操作フローを追記
